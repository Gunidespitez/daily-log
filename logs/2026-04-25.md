## Work log

- built and trained LSTM classifier for adaptive PI controller gain tuning
- dataset had 50% samples at one gain class (Kp=8, Ki=2) — model was just predicting that always
- balanced the dataset, retrained: accuracy dropped from 60% to 34.5%
- 34.5% is the honest number since model is actually trying across all 6 classes now
- root cause identified: only 6 discrete Kp/Ki combinations in training data, input signals (e, de, d, A, r) look too similar across runs so the model can't distinguish them reliably

## What clicked

the 60% wasn't progress, it was the model finding the lazy answer. balancing the data removed that escape route. low accuracy on balanced data is more useful than high accuracy on skewed data — at least now you know what's actually broken.

the real fix isn't the model architecture, it's the data. need continuous Kp/Ki variation from Gautham's simulations instead of 6 fixed points.
