## Work log

* identified core failure in regression model — identical input signals (e, de, d, A, r, d_g) mapped to different Kp/Ki values, causing prediction collapse to mean (~8–9)
* confirmed root cause: model lacked information about **system performance under those gains**
* updated feature set to include overshoot, settling time, and steady-state error (total 9 inputs)
* retrained LSTM with new features + MAE loss + noise injection + sample weighting
* resolved memory issues by subsampling sequences (step=500, ~66k sequences)
* trained final model (v5), achieved stable convergence (val_loss ≈ 0.083)
* validated predictions — model now covers full gain range (Kp: 5–12, Ki: 1–4) instead of collapsing
* generated evaluation plots:

  * predicted vs actual (Kp, Ki)
  * residual analysis
  * before vs after comparison (v4 vs v5)
* fixed compatibility issue between v4 and v5 models (6-feature vs 9-feature input mismatch)
* organized project artifacts and cleaned repository for final version

---

## What clicked

the issue was not model capacity but **data ambiguity** — same system conditions paired with multiple gain values gave the model no basis to choose. adding performance metrics provided context, allowing the model to differentiate between “good” and “bad” gain outcomes for the same input signals.
